# PLANNING.md — 학급운영 플랫폼 설계 문서

## 1. 비전
전국의 교사들이 회원가입 후 5분 안에 학급 페이지를 개설하고,
알림장·일정·시간표를 학생과 공유하며,
개인 업무 공간과 학급운영 도구(자리바꾸기, 마니또, OMR 등)를
한 곳에서 사용하는 학급운영 플랫폼.

## 2. 사용자와 역할
| 역할 | 설명 | 인증 |
|------|------|------|
| teacher | 학급 개설·운영의 주체 | 이메일 가입 |
| student | 학급 페이지 열람, 제출·체크 활동 | 학급코드+번호+PIN |
| parent | (2차 확장) 자녀 관련 열람 | 미정 |
| admin | 서비스 운영자 | 별도 |

## 3. 데이터 구조 초안
```
profiles        — 교사 프로필 (auth.users 연동, role 포함)
academic_years  — 학년도 (교사별, 예: 2026학년도)
classrooms      — 학급 (학년도에 종속, 학급코드, 테마컬러)
students        — 학생 (학급 종속, 번호·닉네임·PIN해시, 실명은 선택)
posts           — 알림장/공지 (읽음확인, 준비물 체크리스트 항목 포함)
events          — 일정 (school | classroom 레이어 구분)
timetables      — 시간표 (요일×교시)
private_notes   — 교사 비공개 메모/할일
student_records — 학생별 기록 카드 (상담·특이사항, 교사만 접근)
tools / tool_results — 도구 실행 결과 (자리배치 등, 공개여부 플래그)
```
모든 테이블에 RLS 정책 필수. student_records와 private_notes는
어떤 경우에도 학생 role이 읽을 수 없어야 한다.

## 4. 화면 목록
### 교사용 (데스크톱 우선)
- 대시보드(투데이 뷰): 오늘 시간표, 오늘 알림장 작성, 할일
- 학급 관리: 명렬 붙여넣기 일괄 등록, 학급코드 발급
- 알림장 작성/읽음현황, 캘린더, 시간표 편집
- 비공개 공간: 메모, 할일, 개인 캘린더, 학생별 기록 카드
- 도구 서랍: 자리바꾸기, 직업바꾸기, 마니또, OMR (결과 공개 토글)

### 학생용 (모바일 우선)
- 홈: 오늘 시간표 + 최신 알림장
- 알림장 목록 + 준비물 체크
- 학급 캘린더, 자리/1인1역 현황판
- (마니또 진행 시) 내 미션 확인

## 5. 마일스톤
### M1 — 수직 조각 (MVP의 MVP)
교사 가입 → 학급 생성 → 명렬 등록 → 알림장 1개 작성 →
학생이 코드로 로그인해서 그 알림장을 본다.
- Supabase 스키마 + RLS (profiles, academic_years, classrooms, students, posts)
- 교사 이메일 인증 / 학생 커스텀 인증
- 최소한의 UI

### M2 — 학급 페이지 완성
- 시간표, 캘린더(학교/학급 레이어), 알림장 읽음확인·준비물 체크
- 학생 홈 투데이 뷰, 교사 대시보드 투데이 뷰

### M3 — 업무 관리 모드 (교사 비공개 공간 확장) ✅ 완료 (2026-07-10)
- 모드 전환: [학급 운영 | 업무 관리] 탭, 마지막 사용 모드 기억
- 업무 데이터는 교사 개인 소유(학년도 무관), 전부 비공개 —
  학생 JWT·공유 링크·anon 어떤 경로로도 접근 불가(RLS 강제)
- 업무 대시보드(오늘 할 일·기한 임박 공문·오늘 일정·자주 쓰는 링크)
- 할 일(우선순위·마감일·매주 반복), 공문 처리 트래커(상태·기한·검색),
  업무 캘린더(학급 일정 겹쳐보기, 공문 기한 자동 표시), 자주 쓰는 링크
- 업무 노트(태그 분류·검색), 문서 스니펫(원클릭 복사),
  학생별 기록 카드(학급 종속·민감정보 — 담임만, 전 경로 RLS 차단)
