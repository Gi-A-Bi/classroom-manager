# PLANNING.md — 학교수첩 설계 문서

> 서비스명: **학교수첩** (2026-07-12 확정, 배포 주소 classroom-manager-gi-a-bis-projects.vercel.app)

## 1. 비전
전국의 교사들이 회원가입 후 5분 안에 학급 페이지를 개설하고,
알림장·일정·시간표를 학생과 공유하며,
개인 업무 공간과 학급운영 도구(자리바꾸기, 마니또, OMR 등)를
한 곳에서 사용하는 학급운영 플랫폼.

## 2. 사용자와 역할
| 역할 | 설명 | 인증 |
|------|------|------|
| teacher | 학급 개설·운영의 주체 | 이메일 가입 |
| student | 학급 페이지 열람, 제출·체크 활동 | 학급코드+번호+PIN |
| parent | (2차 확장) 자녀 관련 열람 | 미정 |
| admin | 서비스 운영자 | 별도 |

## 3. 데이터 구조 초안
```
profiles        — 교사 프로필 (auth.users 연동, role 포함)
academic_years  — 학년도 (교사별, 예: 2026학년도)
classrooms      — 학급 (학년도에 종속, 학급코드, 테마컬러)
students        — 학생 (학급 종속, 번호·닉네임·PIN해시, 실명은 선택)
posts           — 알림장/공지 (읽음확인, 준비물 체크리스트 항목 포함)
events          — 일정 (school | classroom 레이어 구분)
timetables      — 시간표 (요일×교시)
private_notes   — 교사 비공개 메모/할일
student_records — 학생별 기록 카드 (상담·특이사항, 교사만 접근)
tools / tool_results — 도구 실행 결과 (자리배치 등, 공개여부 플래그)
```
모든 테이블에 RLS 정책 필수. student_records와 private_notes는
어떤 경우에도 학생 role이 읽을 수 없어야 한다.

## 4. 화면 목록
### 교사용 (데스크톱 우선)
- 대시보드(투데이 뷰): 오늘 시간표, 오늘 알림장 작성, 할일
- 학급 관리: 명렬 붙여넣기 일괄 등록, 학급코드 발급
- 알림장 작성/읽음현황, 캘린더, 시간표 편집
- 비공개 공간: 메모, 할일, 개인 캘린더, 학생별 기록 카드
- 도구 서랍: 자리바꾸기, 직업바꾸기, 마니또, OMR (결과 공개 토글)

### 학생용 (모바일 우선)
- 홈: 오늘 시간표 + 최신 알림장
- 알림장 목록 + 준비물 체크
- 학급 캘린더, 자리/1인1역 현황판
- (마니또 진행 시) 내 미션 확인

## 5. 마일스톤
### M1 — 수직 조각 (MVP의 MVP)
교사 가입 → 학급 생성 → 명렬 등록 → 알림장 1개 작성 →
학생이 코드로 로그인해서 그 알림장을 본다.
- Supabase 스키마 + RLS (profiles, academic_years, classrooms, students, posts)
- 교사 이메일 인증 / 학생 커스텀 인증
- 최소한의 UI

### M2 — 학급 페이지 완성
- 시간표, 캘린더(학교/학급 레이어), 알림장 읽음확인·준비물 체크
- 학생 홈 투데이 뷰, 교사 대시보드 투데이 뷰

### M3 — 업무 관리 모드 (교사 비공개 공간 확장) ✅ 완료 (2026-07-10)
- 모드 전환: [학급 운영 | 업무 관리] 탭, 마지막 사용 모드 기억
- 업무 데이터는 교사 개인 소유(학년도 무관), 전부 비공개 —
  학생 JWT·공유 링크·anon 어떤 경로로도 접근 불가(RLS 강제)
- 업무 대시보드(오늘 할 일·기한 임박 공문·오늘 일정·자주 쓰는 링크)
- 할 일(우선순위·마감일·매주 반복), 공문 처리 트래커(상태·기한·검색),
  업무 캘린더(학급 일정 겹쳐보기, 공문 기한 자동 표시), 자주 쓰는 링크
- 업무 노트(태그 분류·검색), 문서 스니펫(원클릭 복사),
  학생별 기록 카드(학급 종속·민감정보 — 담임만, 전 경로 RLS 차단)

### M4 — 도구 플랫폼 (방향 변경)
- **결정 (2026-07-10): 학급 도구는 외부 프로그램 링크로 제공한다.**

  1호 내장 도구 자리바꾸기는 중복이라 제거했다.
  - 제거 마이그레이션: 20260710102203_m4_remove_seating_tool.sql
  - 자리바꾸기 등은 별도 외부 프로그램 링크로 안내(업무 관리 모드의 "자주 쓰는 링크"
    또는 추후 학급별 링크 기능 검토).
- (참고) 예전 내장안: 도구 공통 인터페이스 + 자리바꾸기(랜덤 배치·조건·학생 현황판)를

### M5 — 수업 (수업 계획+기록) ✅ 완료 (2026-07-11)
- 업무 관리 모드에 수업 기능: 주간 그리드(요일×교시, 시간표 과목 배경),
  칸별 계획·실행 메모·완료 체크, 지난주 복사·미루기, 과목별 진도표,
  대시보드 오늘 수업 카드. lesson_plans 테이블(담임 전용 RLS, subjects 재사용)
- (구 M5 "확장" 항목이던 학교 그룹·학부모 열람은 아래 로드맵 확산 단계로,
  제출함·성적 개별 피드백은 학기 중 추가 후보로 통합)

## 6. M5 이후 로드맵

### 6-1. 파일럿 패키지 (배포 직후, 우선 순서대로)
1. **배포** — Vercel + 클라우드 Supabase (슬롯 해결 방식은 미정)
2. **PWA** — 학생 폰 홈 화면에 앱처럼 설치 (manifest + 서비스워커)
3. **오늘의 급식** — 나이스 교육정보 개방포털 무료 API 연동.
   학교 설정 후 학생 홈에 자동 표시. 같은 API로 학사일정 자동
   가져오기도 검토
4. **온보딩** — 첫 가입 시 3단계 안내(학급 생성 → 명렬 → 코드 배부)
   + 샘플 학급 체험
5. **주간학습안내 자동 생성** — 수업 주간 그리드 데이터를 학부모용
   주간학습안내 문서로 변환·인쇄

### 6-2. 학기 중 추가 후보
- 알림장 템플릿 저장 + 날짜 예약 게시
- 간단 설문/투표 (체험학습 수요조사, 학급회의 안건 등)
- 전역 검색 (알림장·노트·공문·기록카드 통합 검색)
- 앱 내 "제안하기" 피드백 창구
- 제출함, 성적 개별 피드백 (구 M5 항목)

### 6-3. 학기말~겨울방학 과제
- **생활기록부 시즌 뷰** — 학생 1명 기준 기록카드+성적+출결 통합 화면
- **데이터 내보내기** — 성적·출결·기록카드 일괄 백업 (CSV/엑셀)
- **학년도 전환 마법사** — 새 학년도 시작 버튼 → 작년 아카이브,
  시간표 틀·과목·스니펫 이월, 새 명렬 등록
- **Classup(학급 포인트) 통합** — 플랫폼 정식 기능 재구현 방식으로 겨울방학에
  진행(2026-07-11 분석·계획 수립 완료). Classup DB 동거(스키마 격리+RLS)는
  겨울방학 과제로 연기. 그때 완료되면 배포용 Supabase 프로젝트(별도 계정)를
  본 계정으로 이전하거나 계정 정리 검토

### 6-4. 확산 단계 (여러 학교 사용 시)
- 학교 그룹 — 같은 학교 교사 간 학사일정 공유 (구 M5 항목)
- 스니펫·알림장 템플릿 공개 갤러리 (공유 생태계)
- 학부모 열람 기능 정식화 (구 M5 항목 — 현재는 캘린더 공유 링크만)

### 6-5. 기술 기반 (수시)
- Playwright 회귀 테스트 자산화 — 기능 추가 시마다 기존 기능 자동 점검
- 푸시 알림은 보류 — 웹 푸시 복잡도와 카카오톡 유료 문제로, 당분간
  "매일 아침 확인 습관" 설계(급식+시간표+알림장 한 화면)로 대체

## 7. 보류/추후 결정 사항
- **배포 주소 유지 확정 (2026-07-12)**: classroom-manager-gi-a-bis-projects.vercel.app
  그대로 사용. Vercel 프로젝트 이름 변경(school-note-kr 등)은 하지 않기로 결정 —
  bitly가 이 주소를 가리키고 있어 갱신 연쇄를 피함. 커스텀 도메인 도입 시 재검토.
- 성적 공개 방식: 개별 피드백 채널 vs OMR 일회성 조회 (M4~M5에서 결정)
- 기존 프로그램 원본 형태 확인 후 이식 전략 확정
- 개인정보처리방침 문구 (공개 배포 전 필수)

- 성적 데이터 보관 정책 — 학년도 종료 시 내보내기 안내 후 삭제 예고 방식
  검토(저장 최소화 원칙)
