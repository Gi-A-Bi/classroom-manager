# CLAUDE.md — 학급운영 플랫폼

## 프로젝트 개요
여러 학교의 교사들이 각자 계정으로 학급을 개설·운영하는 멀티테넌트 웹 서비스.
교사는 학급 페이지(공개)와 개인 업무 공간(비공개)을 운영하고,
학생은 학급 코드로 접속해 알림장·일정·시간표 등을 확인한다.

## 기술 스택
- **프레임워크**: Next.js (App Router) + TypeScript
- **백엔드/DB**: Supabase (PostgreSQL + Auth + Row Level Security)
- **배포**: Vercel
- **스타일**: Tailwind CSS, 카드 기반 레이아웃

## 핵심 설계 원칙 (반드시 지킬 것)
1. **테넌트 격리**: 교사는 자기 학급 데이터만, 학생은 자기 학급의 공개 데이터만 접근 가능.
   모든 접근 제어는 앱 코드뿐 아니라 Supabase RLS 정책으로도 이중 강제한다.
2. **학년도 최상위 구조**: 학급은 반드시 학년도(academic_year)에 종속된다.
   매년 3월 새 학년도 전환과 이전 데이터 아카이브를 전제로 설계한다.
3. **개인정보 최소화**: 학생은 번호+닉네임만으로 운영 가능해야 한다.
   생년월일·연락처 등 민감정보 필드는 만들지 않는다.
   성적·상담기록은 저장 최소화 원칙을 따른다.
4. **역할 4단계**: teacher / student / parent(예약) / admin(예약).
   지금 구현하지 않아도 role 체계는 확장 가능하게 둔다.
5. **도구 플러그인 패턴**: 자리바꾸기·마니또·OMR 등 학급 도구는
   공통 인터페이스(명렬 읽기, 결과 쓰기, 학생 공개여부 플래그)를 따르는 모듈로 구현한다.
6. **무료 운영 원칙**: 이 프로젝트는 비용이 청구되어서는 안 된다.
   유료 서비스·유료 API·유료 플랜이 필요한 방식은 무료 대안을 먼저 검토하고,
   유료 요소가 불가피하면 진행하지 말고 사용자에게 먼저 알릴 것.
   외부 서비스 가입은 카드 등록 없이 가능한지 확인할 것.
   유료 전환은 사용자의 명시적 승인 없이 금지.

## 인증 방식
- 교사: 이메일 기반 Supabase Auth
- 학생: 이메일 없이 「학급코드 + 출석번호 + PIN」 커스텀 인증

## 코딩 규칙
- 컴포넌트는 서버 컴포넌트 우선, 필요할 때만 'use client'
- DB 접근은 반드시 RLS가 적용된 클라이언트를 통해서만
- 커밋은 작업 단위별로 작게, 커밋 메시지는 한국어 허용
- UI 텍스트는 전부 한국어
